# App Context

_Generated by app-scout. Static info only — re-check running status each session._

## Recon Summary
Python + Rust (pyo3/maturin via `cotahist_rs`) data pipeline for Brazilian B3 stock exchange historical market data, with SQLite storage, a Streamlit web UI, a backtesting framework (16 strategies), CVM fundamentals pipeline, and an ML feature discovery engine. No Docker or containerization; all services run directly in a Python venv.

## Tech Stack
- **Language:** Python 3.9+, Rust stable (via mise.toml)
- **Runtime:** Python 3.9+, Rust stable toolchain
- **Framework:** Streamlit (UI at `ui/app.py`), pandas/numpy (data), scikit-learn/xgboost/hmmlearn (ML), pytest (testing), pyo3+maturin+rayon (Rust extension)
- **Database:** SQLite — `b3_market_data.sqlite` at project root (no ORM; raw `sqlite3` throughout)
- **Package manager:** pip (`requirements.txt`), cargo (Rust crate in `b3_pipeline_rs/`)

## How to Start the App

Build the Rust extension first (one-time after clone, or after Rust source changes):
```
make dev-rust
```

Then start the Streamlit web UI:
```
streamlit run ui/app.py
```

Or via the convenience launcher:
```
python run_ui.py
```

UI is available at http://localhost:8501.

## Database

**Location:** `b3_market_data.sqlite` at project root. Defined as `DB_PATH` in `b3_pipeline/config.py`.

| Table | Primary Key | Description |
|---|---|---|
| `prices` | `(ticker, date)` | OHLCV + split-adj + adj_close prices |
| `corporate_actions` | `(isin_code, event_date, event_type)` | Dividends, JCP |
| `stock_actions` | `(isin_code, ex_date, action_type)` | Splits, reverse splits, bonus shares |
| `detected_splits` | `(isin_code, ex_date)` | Splits from heuristics |
| `skipped_events` | `(isin_code, event_date, label)` | Unrecognized B3 labels |
| `fetch_failures` | `(company_code, endpoint)` | API fetch failures with retry tracking |
| `cvm_companies` | `cnpj` | CNPJ → ticker + company metadata (listing/delisting dates) |
| `cvm_filings` | `filing_id` | CVM filing index (DFP, ITR, FRE) |
| `fundamentals_pit` | `filing_id` | Point-in-time fundamentals (PIT-safe) |

**Key conventions:**
- `adj_close` — use for backtesting (splits + dividends adjusted); `split_adj_close` for technical signals only
- `volume / 100.0` — COTAHIST financial volume has 2 implied decimal places
- CVM financial values in **thousands of BRL** — multiply by 1000 for market cap comparisons
- Account codes: `3.01` = Revenue, `3.11` = Net Income, `6.01` = Total Assets, `6.02` = Equity
